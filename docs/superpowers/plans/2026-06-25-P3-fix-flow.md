# 核心玩法完整链路实现 Plan

**目标：** 让 P3 核心玩法端到端跑通（导入作品 → 多节点多结局 → 选择 → 转盘 → SSE 故事）

**问题分析：** ParseChain 依赖 LLM API Key，没配置就无法生成节点数据。必须先解决 LLM 不可用时的回退方案。

## Task 1: DemoNodeGenerator — 离线节点生成器

当 LLM 不可用时，生成一个标准的 12 节点多分支多结局图。

**结构：**
```
起点 → 岔路1 → 岔路2 → 岔路3 → 汇合点 → 岔路4 → 最终抉择
  │        │        │        │         │
  └→ 隐藏    └→ 捷径   └→ 风险   └→ 秘密    ├→ 结局1(好)
                                            ├→ 结局2(普通)
                                            ├→ 结局3(坏)
                                            └→ 结局4(隐藏，需属性)
```

- 12 个节点，4 个结局
- 每个节点 3-4 个选项
- 部分选项有属性要求（智力≥60）
- 网状连接，不是线性

## Task 2: ParseChain 回退

当 `callLlm()` 失败时（api-key 为空或网络错误），自动调用 DemoNodeGenerator 生成数据。

## Task 3: 验证端到端流程

Admin 导入（用回退数据）→ Player 选择作品 → 创建会话 → 选选项 → SSE 故事 → 转盘 → 到达结局
