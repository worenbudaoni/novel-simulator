## 任务清单

### 5.1 后端：用户管理 API

- [ ] `GET /api/admin/user/list`（user:read）
  - 用户列表（id、用户名、昵称、角色列表、状态、注册时间）
- [ ] `PUT /api/admin/user/{id}/roles`（user:update_role）
  - 请求体：`{ roleIds: [1, 2, 3] }`
  - 先删除原有 user_role，再批量插入
  - **使对应用户的 Redis 会话失效**（角色变更后旧会话不可用）
- [ ] `PUT /api/admin/user/{id}/status`（user:disable）
  - 启用/禁用用户
  - 禁用后该用户所有会话立即失效

### 5.2 后端：角色管理 API

- [ ] `GET /api/admin/role/list`（role:read）
  - 角色列表（id、编码、名称、描述、是否系统预设）
- [ ] `POST /api/admin/role`（role:manage）
  - 新建自定义角色（系统预设角色不可删除不可编辑）
- [ ] `PUT /api/admin/role/{id}/permissions`（role:manage）
  - 分配权限：勾选的 permission_id 列表
  - 先删除原有 role_permission，再批量插入
  - 使拥有该角色的所有用户 Redis 会话失效

### 5.3 后端：Prompt 配置

- [ ] 实现 `GET /api/admin/settings/prompts` 获取所有 prompt 模板
- [ ] 实现 `PUT /api/admin/settings/prompts` 更新 prompt 模板
- [ ] 每个 Chain（Parse/Story/Branch/Event）的 prompt 可配置
- [ ] prompt 模板存储到数据库或配置文件

### 5.4 前端：Admin 用户管理页

- [ ] 实现 UserManage 页面
  - 用户表格（用户名、昵称、角色标签、状态开关）
  - 分配角色弹窗（多选角色）
  - 启用/禁用开关
  - 搜索/筛选

### 5.5 前端：Admin 角色管理页

- [ ] 实现 RoleManage 页面
  - 角色列表表格
  - 系统预设角色标记（不可删除）
  - 新建角色弹窗
  - 权限分配：按资源分组展示所有权限
  - 全选/取消全选

### 5.6 前端：Admin Prompt 配置页

- [ ] 实现 PromptConfig 页面
  - 四个 Chain 的 prompt 编辑框
  - 每个 prompt 支持 `{{变量}}` 占位提示
  - 保存按钮

### 5.7 前端：Player 移动端适配

## 验收标准

1. Admin 可在界面中管理用户（分配角色、启用/禁用）
2. Admin 可在界面中管理角色（新建角色、分配权限）
3. Admin 可配置四个 Chain 的 Prompt 模板
4. Player 在手机浏览器上核心功能可用
5. Admin 在手机上基础可用
6. 所有组件覆盖 loading/empty/error 状态
7. 角色/权限变更后用户会话正确失效
