# P3 核心玩法

> 对应总设计: `../specs/2026-06-18-novel-interactive-story-simulator-design.md`
> 预估工时: ~3 周
> 验收标准: 用户可以选择作品、探索节点地图、做出选择/转盘抽奖、LLM 生成故事、存档读档完整跑通

---

## 本阶段目标

完成 Player 端的核心游戏体验，包括作品选择、节点地图探索、选项面板、转盘抽奖、LLM 事件和故事生成、SSE 流式输出以及存档功能。

---

## 依赖条件

- P1 已完成（Auth + 数据库 + 前端框架）
- P2 已完成（至少有 1 部导入完成的小说可用于测试）

## 任务清单

### 3.1 后端：Player 作品列表

- [ ] 实现 `GET /api/player/novel/list`（player:play）
  - 根据当前用户的角色列表，查询 novel_role_visibility
  - 返回当前用户可见的作品列表
  - GUEST 用户返回 GUEST 角色可见的作品
- [ ] 实现 `GET /api/player/novel/{novelId}/full`（player:play）
  - 返回完整节点树（节点、边、选项）
  - 校验当前用户是否有权访问该作品

### 3.2 后端：游戏会话（存档）

- [ ] 实现 `POST /api/player/session/create`（player:play）
  - 创建游戏存档（user_id = 当前用户 id，GUEST 为 NULL）
  - 初始化节点状态（标记起始节点）
  - 初始化角色属性（user_character 默认值）
  - 返回 sessionId
- [ ] 实现 `GET /api/player/session/{id}`（player:play）
  - 获取当前存档状态（当前位置、历史路径、属性等）
- [ ] 实现 `POST /api/player/session/save`（player:play）
  - 手动保存当前进度到 MySQL
- [ ] 实现 `POST /api/player/session/load`（player:play）
  - 从 MySQL 加载指定存档
- [ ] 实现 `GET /api/player/session/{id}/saves`（player:play）
  - 列出该游戏会话的所有存档点
- [ ] 实现 `POST /api/player/session/restart`（player:play）
  - 重置到起点

### 3.3 后端：节点引擎

- [ ] 实现 `GET /api/player/node/{nodeId}`（player:play）
  - 返回节点详情 + 该节点的选项列表
  - 选项包含 label、target_node_id（NULL=LLM动态生成）、trigger_event、risk_hint
- [ ] 实现 `POST /api/player/action/choose`（player:play）
  - 请求体：`{ sessionId, optionId }`
  - 记录选择到 history_path
  - 如果 option.trigger_event = true，按用户设置的概率判定是否触发事件
  - 如果 option.target_node_id 为 NULL，调用 BranchChain 生成动态分支
  - 更新存档位置
  - 返回结果（事件内容/新节点信息），后续由前端触发故事流

### 3.4 后端：转盘抽奖

- [ ] 实现 `POST /api/player/action/spin`（player:play）
  - 请求体：`{ sessionId, nodeId }`
  - 调用 EventEngine 从事件池按权重抽取事件
  - 考虑用户设置的概率和死亡率
  - 如果事件池不足，调用 EventChain 动态生成
  - 标记该事件已使用（避免重复）
  - 事件结果包含：title、content、event_type、death_probability
  - 如果抽到死亡事件且判定死亡 → 标记角色死亡
  - 返回事件结果

### 3.5 后端：事件引擎

- [ ] 实现 EventEngine Service
  - getEventPool(nodeId)：从 random_event 表取该节点事件 + 全局事件
  - 缓存到 Redis（cache:events:{novelId}:{nodeId}）
  - weightedRandom(pool)：按权重抽取
  - 考虑用户设置的随机率（决定是否触发）
  - 考虑用户设置的死亡率（负面事件中判定）
  - markUsed(sessionId, eventId)：避免重复使用

### 3.6 后端：故事生成 + SSE 流式输出

- [ ] 实现 `GET /api/player/story/stream/{sessionId}`（player:play）
  - 使用 SSE（SseEmitter）流式输出
  - 触发时机：用户在 choose/spin 后主动调用该接口
- [ ] 实现 StoryChain
  - 输入：世界观 + 历史路径 + 当前节点 + 选择/事件结果 + 角色属性
  - 输出：流式 Markdown 文本
  - 流式输出时附带生成 story_summary（300 字以内摘要）
- [ ] 实现 EventChain
  - 输入：世界观 + 当前节点 + 小说重要节点走向 + 历史路径 + 结果类型
  - 输出：事件描述 + 属性影响 JSON
- [ ] 实现 BranchChain
  - 输入：世界观 + 当前节点 + 用户历史 + 角色属性
  - 输出：新节点 + 选项 + 关联事件
- [ ] 实现 LLMFactory（运行时切换 LLM 配置）
  - 默认：application.yml 配置
  - 用户设置覆盖：根据 session 的 settings_json 动态创建 OpenAiChatModel

### 3.7 后端：用户设置

- [ ] 实现 `POST /api/player/session/settings`（player:play）
  - 保存随机率、死亡率、LLM 配置到 Redis（cache:session:{id}:settings）
  - 冒险开始后锁定（保存时检查是否已开始）

## 验收标准

1. 用户能看到自己角色可见的作品列表
2. 从起点节点开始，每个节点展示选项
3. 选择后 SSE 流式生成故事段落
4. 转盘抽奖可正常触发，事件符合世界观
5. 死亡判定正常（按配置的死亡率）
6. 存档/读档正常
7. GUEST 和已登录用户体验差异正确
8. LLM 运行时切换生效
