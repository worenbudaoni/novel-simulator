# P1 基础架构

> 对应总设计: `../specs/2026-06-18-novel-interactive-story-simulator-design.md`
> 预估工时: ~2 周
> 验收标准: 前后端项目启动成功，注册登录跑通，数据库表创建完成

---

## 本阶段目标

搭建项目骨架，完成用户认证和 RBAC 权限基础设施，确保后续阶段能在此基础上直接开发功能。

---

## 任务清单

### 1.1 后端项目初始化

- [x] Spring Boot 2.6.13 项目已存在（基础骨架）
- [ ] 引入 MyBatisPlus 依赖，配置数据源
- [ ] 引入 Spring Security 依赖（用于 BCrypt 和 @PreAuthorize）
- [ ] 引入 LangChain4j 依赖（空壳，本阶段不接入 LLM）
- [ ] 配置 MySQL 连接（application.yml）
- [ ] 配置 Redis 连接（application.yml）
- [ ] 配置 Maven 依赖（pom.xml）：spring-boot-starter-web、mybatis-plus-boot-starter、spring-boot-starter-security、spring-boot-starter-data-redis、mysql-connector-java

### 1.2 数据库建表

- [ ] 执行 DDL 创建以下表：

```sql
-- user（用户表）
-- role（角色表）
-- permission（权限表）
-- user_role（用户-角色关联）
-- role_permission（角色-权限关联）
-- novel（小说表）
-- novel_role_visibility（作品-角色可见性）
-- node（核心节点表）
-- node_edge（节点连接表）
-- node_option（节点选项表）
-- random_event（随机事件表）
-- user_session（用户存档表）
-- user_character（角色属性表）
-- parse_record（LLM解析记录表）
-- llm_cache（LLM 生成缓存）
```

- [ ] 插入初始数据：
  - 4 个预设角色：ADMIN、EDITOR、USER、GUEST
  - 全部预设权限（参见总设计 3.3 权限清单）
  - ADMIN 角色分配全部权限
  - EDITOR 角色分配：novel:read, novel:update, node:*, event:*, role:read
  - USER 角色分配：novel:read, node:read, event:read, player:play, player:save, player:spin
  - GUEST 角色分配：novel:read, node:read, event:read, player:play, player:save, player:spin
  - 创建一个默认管理员账号（admin / admin123）

### 1.3 后端 Entity + Mapper

- [ ] 为每张表创建 Entity 类
- [ ] 为每张表创建 Mapper 接口（MyBatisPlus BaseMapper）
- [ ] 配置 MyBatisPlus 分页插件

### 1.4 用户认证（Auth）

- [ ] 实现 `POST /api/auth/register` 用户注册
- [ ] 实现 `POST /api/auth/login` 用户登录
  - BCrypt 验证密码
  - 查询 user_role 获取用户角色列表
  - 查询 role_permission 获取权限列表
  - 生成 UUID sessionId，存入 Redis
- [ ] 实现 `POST /api/auth/logout` 退出登录
- [ ] 实现 `GET /api/auth/me` 获取当前用户信息
- [ ] 实现 AuthFilter
  - 从 Authorization header 提取 sessionId
  - 查询 Redis 验证会话
  - 设置 SecurityContext（含 roles 和 permissions）
  - 白名单路径: /api/auth/login, /api/auth/register
- [ ] 实现 Redis 会话续期（每次请求自动续 24h TTL）

### 1.5 角色/权限加载

- [ ] 实现 UserDetailsService：从 user_role → role_permission 加载权限
- [ ] 将权限列表以 GrantedAuthority 形式注入 SecurityContext
- [ ] 实现一个测试接口 `GET /api/test/auth` 验证 @PreAuthorize 生效

### 1.6 前端项目初始化

- [ ] 使用 Vite 创建 React + TypeScript 项目
- [ ] 安装 shadcn/ui + Tailwind CSS
- [ ] 配置 Tailwind 主题色（可适配小说阅读氛围）
- [ ] 安装 react-flow（节点图组件）
- [ ] 安装 react-markdown + remark-gfm（MD 渲染）
- [ ] 创建基础路由结构（react-router-dom）
  - /login、/register
  - /player/*、/admin/*
- [ ] 实现 useAuth hook（管理登录状态、sessionId 存储）
- [ ] 实现 API client（axios 封装，自动携带 Authorization header）
- [ ] 实现 ProtectedRoute 组件（检查当前用户是否有指定权限）
- [ ] 实现 LoginPage、RegisterPage 页面
- [ ] 实现 Navbar 组件（登录状态、用户菜单）
- [ ] 实现 App.tsx 路由配置

### 1.7 联调验证

- [ ] 注册 → 登录 → 获取用户信息 → 退出 → 会话失效
- [ ] 未登录请求需登录接口 → 返回 401
- [ ] 无权限请求 Admin 接口 → 返回 403
- [ ] 不同角色登录验证权限正确

## 验收标准

1. 前后端启动成功，无报错
2. 用户注册登录流程完整，Redis 会话正常
3. AuthFilter 正确拦截未登录请求
4. @PreAuthorize 正确拦截无权限请求
5. 所有数据库表创建成功，初始数据正确
6. 前端路由正常跳转，登录状态全局共享
