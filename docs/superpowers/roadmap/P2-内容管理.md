# P2 内容管理

> 对应总设计: `../specs/2026-06-18-novel-interactive-story-simulator-design.md`
> 预估工时: ~1.5 周
> 验收标准: Admin 可以完整导入一部小说（TXT/联网搜索/LLM 解析），管理节点和事件

---

## 本阶段目标

完成 Admin 小说管理功能，包括作品 CRUD、TXT 上传导入、联网搜索导入、LLM 解析、节点编辑器、事件池管理、可见角色配置。

---

## 依赖条件

- P1 已完成
- 数据库表已创建
- AuthFilter + @PreAuthorize 可用
- 已有管理员账号

---

## 任务清单

### 2.1 后端：作品 CRUD

- [ ] 实现 NovelController（全部标注 @PreAuthorize）
  - `POST /api/admin/novel` — 新建作品（novel:create）
  - `GET /api/admin/novel/list` — 作品列表（novel:read）
  - `GET /api/admin/novel/{id}` — 作品详情（novel:read）
  - `PUT /api/admin/novel/{id}` — 修改作品（novel:update）
  - `DELETE /api/admin/novel/{id}` — 删除作品（novel:delete）
- [ ] 实现 NovelService（MyBatisPlus CRUD）
- [ ] 实现 `PUT /api/admin/novel/{id}/visibility` — 设置可见角色（novel:set_visibility）
  - 请求体：`{ roleIds: [1, 2, 3] }`
  - 先删除原有记录，再批量插入 novel_role_visibility

### 2.2 后端：TXT 导入

- [ ] 实现 `POST /api/admin/novel/import/upload`（novel:create）
  - 接收 MultipartFile（.txt）
  - 读取文件内容，存入 novel.raw_content
  - 调用 ParseChain 解析
- [ ] 实现 ParseChain（第一个 LangChain4j Chain）
  - 输入：小说 TXT 内容
  - Prompt：输出结构化 JSON（世界观/节点/关系/事件/属性模板）
  - 输出：解析后的 JSON
  - 注意：本阶段先同步调用，后续可改为异步

### 2.3 后端：联网搜索导入

- [ ] 实现 `POST /api/admin/novel/import/search`（novel:create）
  - 请求体：`{ name: "三体", type: "NOVEL" }`
  - 通过 LangChain4j 检索工具联网搜索
  - 将搜索结果喂给 ParseChain 生成框架
- [ ] 实现 `POST /api/admin/novel/import/check`（novel:create）
  - 检查某作品是否能搜到足够信息
  - 返回：`{ exists: true/false, summary: "..." }`
- [ ] 不足信息的统一返回提示

### 2.4 后端：节点/事件管理

- [ ] 实现 NodeController
  - `PUT /api/admin/novel/{id}/nodes` — 批量保存/更新节点（node:update）
  - 接受节点数组、节点关系数组
- [ ] 实现 EventController
  - `PUT /api/admin/novel/{id}/events` — 批量保存/更新事件（event:update）
- [ ] 实现对应 Service

### 2.5 后端：LLM 解析记录

- [ ] 解析成功后，将结构化 JSON 写入对应表（node / node_edge / node_option / random_event）
- [ ] 写入 parse_record（记录原始响应、tokens 消耗）
- [ ] 写入 llm_cache（缓存本次解析结果）

### 2.6 前端：Admin 作品管理页

- [ ] 实现 NovelList 页面
  - 作品表格（标题、作者、类型、状态、可见角色标签）
  - 新建/删除操作
- [ ] 实现 NovelImport 页面
  - 输入作品名称
  - 选择类型（小说/动漫/漫画）
  - 选择导入方式（TXT 上传 / 联网搜索）
  - TXT 上传：文件选择器 + 上传按钮
  - 联网搜索：输入名称 → 搜索
  - 解析进度状态（loading / preview / error）
  - 解析结果预览（世界观摘要、节点列表、事件列表）
  - 配置可见角色（多选角色）
  - 确认导入按钮

### 2.7 前端：Admin 节点/事件管理页

- [ ] 实现 NodeEditor 页面（React Flow）
  - 使用 React Flow 渲染节点关系图
  - 拖拽添加新节点
  - 编辑节点标题、描述
  - 编辑节点连接线
  - 配置节点选项
- [ ] 实现 EventPool 页面
  - 事件表格（标题、类型、权重、关联节点）

### 2.8 联调验证

- [ ] 完整走一遍：新建小说 → TXT 上传 → LLM 解析 → 预览 → 确认导入 → 数据库正确
- [ ] 联网搜索导入流程正常
- [ ] 搜索不到的作品返回正确提示
- [ ] 节点编辑器能正常拖拽和保存
- [ ] 可见角色配置生效

---

## 验收标准

1. Admin 可以完整导入一部小说（TXT 或联网搜索）
2. LLM 解析结果正确写入所有相关表
3. 搜索不到的作品给出明确提示
4. Admin 可在节点编辑器中拖拽编辑节点关系
5. Admin 可管理事件池
6. 可见角色配置生效（后续 Player 端按角色过滤）
