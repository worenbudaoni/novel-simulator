## 本阶段目标

完成角色属性系统、称号系统、结局评分与评级、LLM 动态分支生成、故事长度管理与 Token 压缩。

---

## 依赖条件

- P3 已完成（核心玩法跑通）

---

## 任务清单

### 4.1 后端：角色属性系统

- [ ] 在 EventEngine 中集成属性变化逻辑
  - 事件结果附带 attr_changes（如 `{"hp": -30, "attack": +5}`）
  - 事件触发后更新 user_character
- [ ] 在 BranchChain 中集成属性变化逻辑
  - LLM 生成分支节点时，可根据用户历史调整属性
- [ ] 在 ActionEngine 中集成属性变化
  - 某些选项本身可带属性变化（如 "冒险跳过悬崖" → hp-20）
- [ ] 属性缓存到 Redis（cache:session:{id}:character）
- [ ] 属性变化推送前端（SSE 附带属性更新事件）

### 4.2 后端：称号系统

- [ ] 设计称号条件和触发逻辑
  - 首次做出选择 → 「初出茅庐」
  - 连续选对 3 次 → 「明智抉择」
  - 触发 5 次事件 → 「命运之子」
  - 死亡 1 次 → 「向死而生」
  - 全部正面选项 → 「完美主义者」
- [ ] 在后端的 节点引擎/事件引擎 中嵌入称号检查
- [ ] 称号获得时，通过 SSE 推送给前端

### 4.3 后端：结局评分

- [ ] 实现评分 Service
  - 基础分 200
  - 存活节点数 × 10
  - 获得称号数 × 50
  - 触发事件数 × 15
  - 死亡次数 × -80
  - LLM 评价分（0-200）
  - 故事完整度（0-100）
- [ ] 评级映射：SSS ≥ 950 / S ≥ 800 / A ≥ 650 / B ≥ 500 / C ≥ 300 / D < 300
- [ ] LLM 评价分：调用 StoryChain 或单独的 JudgeChain 生成
- [ ] 到达结局节点或死亡时自动触发评分

### 4.4 后端：动态分支节点（BranchChain 增强）

- [ ] BranchChain 实现
  - 输入：世界观 + 当前节点 + 用户历史 + 角色属性
  - 输出：新节点（title, description）+ 选项 + 关联事件
- [ ] 生成的节点和事件写入数据库（node_type = 1 标记 LLM 生成）
- [ ] 生成的节点加入节点树缓存
- [ ] 分支节点永久保存，后续用户也可能遇到

### 4.5 后端：故事长度管理

- [ ] StoryChain 每次生成时附带输出 story_summary（300 字以内）
- [ ] 生成上下文策略：
  - 早期（历史短）：完整历史 + 当前
  - 后续：story_summary + 最近 3 次交互 + 角色属性 + 世界观
- [ ] story_summary 存入 user_session.story_summary

### 4.6 后端：LLM 结果缓存

- [ ] 实现 LLM 缓存检查：相同输入不重复调用
- [ ] cache key = 输入内容的 MD5 哈希
- [ ] 存储到 llm_cache 表 + Redis（cache:llm:{hash}）
- [ ] TTL：Redis 7 天，MySQL 长期保存
- [ ] 缓存命中直接返回，不调用 LLM

### 4.7 前端：结局页面

- [ ] 实现 StoryEnding 页面
  - 角色信息卡（称号 + 六维属性）
  - 评分结果（评级徽章 + 分数 + 数据统计）
  - LLM 评价文字
  - 完整故事全文（Markdown 渲染）
  - 操作按钮：再来一次 / 导出故事 / 返回作品选择

### 4.8 前端：故事取回接口

- [ ] 实现 `GET /api/player/session/{id}/ending`（player:play）
  - 返回结局完整数据（角色信息、评分、全文、统计）

### 4.9 前端：属性/称号动画

- [ ] 属性变化动画提示（+5/-10 浮动文字效果）
- [ ] 获得称号时弹出通知
- [ ] CharacterPanel 实时更新

### 4.10 联调验证

- [ ] 完整走一遍：开始冒险 → 选择/事件 → 属性变化 → 获得称号 → 到达结局 → 评分 → 结局页面
- [ ] 属性变化准确反映到数据库和前端
- [ ] 称号条件正确触发
- [ ] 评分公式计算正确
- [ ] LLM 生成的动态分支节点合理
- [ ] 结果缓存命中时不再调用 LLM

---

## 验收标准

1. 角色六维属性在冒险中动态变化
2. 称号在满足条件时自动获得
3. 结局时正确评分和评级
4. LLM 动态生成的分支节点符合世界观
5. 故事摘要压缩正常工作
6. LLM 结果缓存减少重复调用
